# Package: src.sentrymode.monitoring

1. Status: Shared monitoring kernel used by all factors and CLI flows.
2. Logic: Defines runtime settings, factor/notifier protocols, result/context models, and orchestration/report rendering.
3. Constraints:
   - Stay factor-agnostic; do not import concrete modules from `sentrymode.factors`.
   - Keep external notification side effects behind notifier implementations.
   - Preserve typed interfaces (`Factor`, `Notifier`) as the extension seam for new signal modules.

## Member List

- `__init__.py`: public API exports for monitoring primitives.
- `models.py`: protocols and dataclasses (`FactorResult`, `MonitorContext`, `Severity`).
- `settings.py`: `pydantic-settings` runtime configuration schema and validators.
- `runner.py`: scheduling loop, factor execution isolation, and report aggregation.
- `notifiers.py`: console and Bark notification adapter.
